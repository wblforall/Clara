Analisis Sistem & Potensi Celah
4.1 Analisis: Ketiadaan Fungsi "Restore UI" pada Modul Transaksi Dihapus
Halaman ?r=deleted_transactions menampilkan daftar transaksi yang telah dihapus dengan mekanisme
soft-delete (kolom deleted_at di tabel MySQL), namun tidak menyediakan tombol atau aksi UI untuk
memulihkan (restore) data tersebut.
🔴 Dampak Operasional terhadap Integritas Database clara_unified
ASPEK RISIKO 	DETAIL DAMPAK
Pemulihan
Manual SQL
Tim IT harus menjalankan UPDATE transactions SET deleted_at = NULL WHERE id =
X secara manual, meningkatkan risiko human error pada query production database.
Inkonsistensi
Data Terkait
Transaksi berelasi dengan tabel daily_allocations , skp_entries , dan
pic_achievements . Restore tanpa logika bisnis yang benar dapat menciptakan double-
counting pada laporan achievement PIC.
Audit Trail Rusak 	Restore manual SQL tidak menciptakan entri di tabel Activity Log, sehingga aksi tersebut
tidak traceable — melanggar prinsip audit accountability.
Downtime
Operasional
Setiap kesalahan hapus memerlukan eskalasi ke tim developer/DBA. Dalam lingkungan
mall dengan transaksi real-time, ini menciptakan bottleneck operasional yang kritis.
🚨 Penilaian Kritis
Ketiadaan Restore UI bukan sekadar fitur yang hilang — ini adalah kerentanan integritas data (data
integrity vulnerability). Sistem yang menawarkan soft-delete tanpa UI restore secara arsitektural
incomplete dan tidak layak produksi.
4.2 Analisis: Kegagalan Arsitektur TV Display (Token Berbasis URL)
Modul Display TV menggunakan pendekatan URL-based token authentication: ?r=display&token=tv-token-
12345 . Jika token kosong atau tidak ada di URL, sistem langsung crash dengan error fatal tanpa fallback.

Audit Temuan Cacat Sistem (True Defects)
🔴 DEF-001: Halaman Transaksi Dihapus Tidak Memiliki Tombol/Aksi
Restore
HIGH — Kritis
Lokasi: /?r=deleted_transactions
Deskripsi: Fitur soft-delete pada transaksi tersedia dan berfungsi — transaksi yang dihapus masuk ke
halaman ini dengan kolom deleted_at terisi. Namun, antarmuka tidak menyediakan tombol "Pulihkan" /
"Restore" apapun. Halaman bersifat read-only (tabel statis).
Repro Steps:
1. Login sebagai Superadmin.
2. Hapus sembarang transaksi dari daftar transaksi aktif.
3. Buka /?r=deleted_transactions .
4. Amati: tidak ada tombol restore/pulihkan pada baris manapun.
Dampak: Superadmin tidak dapat memulihkan kesalahan penghapusan secara mandiri. Pemulihan hanya
bisa dilakukan via DML SQL manual oleh Database Administrator, menciptakan bottleneck, risiko human
error, dan ketidakterlacakan aksi restore di Activity Log.
🟠 DEF-002: TV Display Crash Fatal Saat Akses Tanpa Parameter Token 	MEDIUM
Lokasi: /?r=display (tanpa parameter token )
