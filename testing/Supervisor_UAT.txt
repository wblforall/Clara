Analisis Sistem & Potensi Celah (System Analysis)
Berdasarkan audit mendalam, ditemukan dua kerentanan fungsional/arsitektur yang signifikan. Meskipun
modul ini diperuntukkan bagi Superadmin/Admin, dampaknya beresonansi pada integritas operasional
secara keseluruhan.
🔴 1. Celah Integritas Data: Tidak Ada Tombol "Restore" di Halaman
Transaksi Dihapus
HIGH — KRITIS
Lokasi: /?r=deleted_transactions
Deskripsi Masalah: Sistem CLARA telah mengimplementasikan konsep Soft Deletes di level database (kolom
deleted_at ), dan transaksi yang dihapus berhasil dipindahkan ke halaman "Transaksi Dihapus". Namun,
secara UI/UX, tidak disediakan aksi atau tombol untuk "Memulihkan" (Restore) data tersebut.
Dampak Analisis Sistem:
Bottleneck Operasional: Jika terjadi human error (salah hapus), pengguna tidak dapat membatalkannya. Tim
IT/Database Administrator harus dihubungi untuk menjalankan kueri SQL DML manual ( UPDATE
transactions SET deleted_at = NULL ).
Risiko Audit: Pemulihan manual melalui SQL tidak akan tercatat dalam Activity Log (Log Audit) aplikasi,
sehingga memutus rantai keterlacakan (traceability).
Bukti Visual Defect-01: Halaman transaksi terhapus hanya berupa tabel baca (read-only) tanpa aksi perbaikan.
🟠 2. Kerentanan Stabilitas: TV Display Crash Fatal Tanpa Token Valid 	MEDIUM

Lokasi: /?r=display
Deskripsi Masalah: Halaman TV Display (yang biasanya ditampilkan di monitor publik mall) menggunakan
validasi token berbasis URL ( ?token=xxx ). Jika URL diakses tanpa token atau token kadaluarsa, aplikasi
memunculkan Fatal Error "Display token tidak valid" secara vulgar di layar, tanpa mekanisme graceful
degradation atau fallback.
Dampak Analisis Sistem:
Risiko Reputasi: Layar publik di mal yang menampilkan stack trace error atau pesan error fatal merusak citra
profesional.
Kelemahan Arsitektur URL Token: Bergantung pada parameter URL rentan terhadap auto-refresh browser
yang menghapus query string. Disarankan migrasi ke autentikasi berbasis sesi (cookie) yang persisten khusus
untuk perangkat display.
Bukti Visual Defect-02: Fatal error (crash code Laravel) saat mengakses modul TV Display tanpa parameter
kredensial URL.
