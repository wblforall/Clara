1. Pendahuluan
Laporan ini disusun untuk mendokumentasikan hasil pengujian fungsionalitas end-to-end (E2E) dan
kepatuhan sistem keamanan pada aplikasi CLARA (Casual Leasing Achievement & Revenue Analytics).
Pengujian dilakukan menggunakan akun superadmin utama saadilaeffendi@gmail.com dengan tujuan
memvalidasi alur kerja administrasi, manajemen pengguna, pengaturan hak akses peran, data master
exhibition, pic, target penjualan, manajemen client, log audit, serta memeriksa celah keamanan akses
langsung ke berkas sensitif di server.
Seluruh skenario pengujian dieksekusi secara otomatis menggunakan Puppeteer headless browser di
lingkungan lokal. Hasil capture layar (screenshot) disematkan pada setiap langkah untuk memverifikasi
kesesuaian visual dan perilaku sistem.
Informasi Lingkungan Pengujian:
• URL Aplikasi: http://localhost:8000
• Database: clara_unified (MySQL di XAMPP)
• Akun Uji: saadilaeffendi@gmail.com / pok3mon (Superadmin)

12. Pengujian Celah Keamanan Akses Berkas Langsung
(Direct File Access)
Pengujian keamanan difokuskan pada perlindungan berkas tanda tangan digital dan lampiran dokumen
penawaran. Sistem wajib memverifikasi kepemilikan sesi sebelum melayani pengunduhan berkas sensitif
dari direktori signatures/.
1. Membuka peramban baru tanpa sesi login aktif (tanpa cookie).
2. Mengakses URL penjarahan dokumen secara langsung: http://localhost:8000/?
r=file&file=signatures/test.png.
3. Memastikan server menolak akses dengan respons HTTP 403 Forbidden atau mengalihkan ke
gerbang login.
Gambar 12.1: Bukti penolakan server (HTTP 403 / Halaman Kosong/Blocked) ketika diakses langsung tanpa sesi.
Analisis Keamanan: Mekanisme autentikasi pada rute ?r=file berjalan dengan baik. Server memverifikasi
keberadaan session cookie sebelum membaca isi disk, mencegah kebocoran dokumen oleh pihak eksternal yang
tidak memiliki otorisasi.
